# Tracer для широкой аудитории

Эта страница помогает объяснить инструмент команде, стейкхолдерам и новым разработчикам без погружения в детали API.

## 1) Что это в одной фразе

`Tracer` показывает реальный runtime-путь выполнения: кто кого вызвал, в каком порядке и где возникла проблема.

## 1.1) Что такое `slice` в терминах Tracer

Точное определение:

- `trace` в Tracer — это проекция потока вызовов (call stack) в виде событий;
- `slice` — это часть call stack, у которой есть начало и конец;
- то есть `slice` — это отрезок на call stack.

Как задать слайс на практике:

- `predicate` задает правило выбора функции (или группы функций), для которой строится отрезок;
- `beforeCall` срабатывает на входе в вызов функции, выбранной `predicate`; возврат `true` запускает отрезок;
- `afterCall` срабатывает на выходе из вызова этой функции; возврат `false` завершает отрезок.

Мини-пример:

```js
Tracer.defineSlice("ConvertToPdf", {
  predicate: (e) => e.fnKey === "asc_nativeGetPDF",
  beforeCall: () => true,   // начало отрезка
  afterCall: () => false    // конец отрезка
});

Tracer.traceBySlice("ConvertToPdf", (event) => {
  console.log(event.fullName);
});
```

Что это значит в рантайме:

- в общий `trace` попадают все события;
- в `traceBySlice("ConvertToPdf", ...)` попадают только события между началом и концом этого отрезка.

## 2) Какую боль решает

- В легаси-коде баги часто "теряются" между модулями.
- Статический просмотр кода не показывает реальный сценарий исполнения.
- Команда тратит много времени на догадки и ручной поиск.

Что меняется с Tracer:

- вместо догадок есть фактическая цепочка вызовов;
- проще локализовать регрессию и рекурсию;
- быстрее объяснять проблему коллегам и ИИ-ассистенту.

## 3) 3-минутное демо для презентации

### Шаг 1. Включить узкий сценарий (создать `slice`)

```js
Tracer.defineSlice("ConvertToPdf", {
  predicate: (e) => e.fnKey === "asc_nativeGetPDF",
  beforeCall: () => true,
  afterCall: () => false
});
```

### Шаг 2. Подписаться только на нужный отрезок

```js
Tracer.traceBySlice("ConvertToPdf", (event) => {
  console.log(event.fullName, event.callStack);
});
```

### Шаг 3. Запустить проблемный сценарий и показать цепочку

Покажите 5-10 строк лога и объясните:
- где начинается поток;
- какой вызов повторяется;
- где возникает ошибка.

## 4) Quick Start (5 минут)

### Вариант A: ручное подключение Tracer

```js
Tracer.setTraceProfile("balanced");
Tracer.traceCalls((event) => {
  if (event.place === "before") {
    console.log("->", event.fullName);
  }
});
```

### Вариант B: автоподключение через `webpack-tracer-plugin`

```js
const { UniversalCodeInjectorPlugin } = require("webpack-tracer-plugin");

plugins: [
  new UniversalCodeInjectorPlugin({
    injectTracerRuntimeFirst: true,
    injectLoaderOpts: {
      targets: ["UserService", "OrderService"],
      generateCode: {
        construct: ({ className }) =>
          `Tracer.observeProperty(this, "state", "${className}");`
      }
    }
  })
];
```

## 5) Как объяснять ценность для AI

- ИИ точнее анализирует проблему, когда видит не только код, но и runtime-трейс.
- Tracer дает структурированные события (`fullName`, `fnKey`, `callStack`, `args`).
- Это снижает число ложных гипотез и ускоряет точечные фиксы.

Короткая формула:

`Код + Runtime-трейс + Контекст ошибки = более точные ответы ИИ`

## 6) Когда применять, а когда нет

Применять:
- большой legacy без полной документации;
- сложные интеграционные баги;
- регрессии после изменений в нескольких модулях.

Не применять как основной режим:
- везде и всегда в production без фильтров;
- при отсутствии узкого сценария (будет много шума).

## 7) FAQ для широкой аудитории

### Это APM?
Нет. Это инженерный runtime-инструмент для точечной диагностики и исследования поведения кода.

### Это замедляет приложение?
Да, при широком покрытии. Поэтому используйте узкие `slice`, фильтры и профиль `balanced`.

### Подходит только для экспертов?
Нет. Базовый сценарий можно запустить за 5 минут по шаблону выше.
